## 🔄 ЛОГИКА РАБОТЫ (алгоритм)

### При регистрации / смене группы:

```
1. Пользователь вводит группу
2. Бот проверяет: существует ли группа в БД?
3. Если да:
   а) Удаляем ВСЕ старые подгруппы пользователя (если были)
   б) Находим ВСЕ уникальные предметы для этой группы из расписания
   в) Добавляем каждый предмет с подгруппой = NULL (основная группа)
   г) Сохраняем в БД
   д) Предлагаем пользователю ввести предметы с отклонениями
4. Если нет:
   а) Сообщаем об ошибке
   б) Показываем список доступных групп
   в) Просим ввести заново
```

### При добавлении предмета пользователем:

```
1. Пользователь вводит название предмета
2. Бот проверяет:
   а) Есть ли такой предмет в расписании?
   б) Есть ли уже такой предмет у пользователя?
3. Если предмет есть в расписании и ещё не добавлен пользователю:
   а) Добавляем предмет с подгруппой = NULL (основная группа)
   б) Предлагаем ввести подгруппу, если она отличается
4. Если предмет уже есть у пользователя:
   а) Сообщаем об этом
   б) Предлагаем редактировать
```

### При редактировании предмета:

```
1. Пользователь выбирает предмет из своего списка
2. Бот показывает текущую подгруппу
3. Пользователь вводит новую подгруппу или "Основная"
4. Бот проверяет: существует ли подгруппа для этого предмета в БД?
5. Если да — обновляет
6. Если нет — сообщает об ошибке и показывает доступные подгруппы
```
